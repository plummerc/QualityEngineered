Where do you get stuck? We can help!

Requirements Analysis Test Planning Test Case Development Environment Setup Test Execution Test Closure

Enlist our services for assistance within the remit of any stage (or all stages) of the software testing life cycle with regards to white box testing, gray box testing, and black box testing.

Requirements Analysis

In this phase, testers analyze the customer requirements and work with developers during the design phase to see which requirements are testable and how they are going to test those requirements.
It is very important to start testing activities from the requirement phase itself, because the cost of fixing bugs is significantly reduced if they are found in the requirements phase.

Test Planning

In this phase, testers are typically finding answers to the following questions:

  • What are the risks being mitigated by testing?
  • What needs to be tested?
  • How will testing be done?
  • What test strategy will be followed?
  • What will be the testing environment?
  • Which test methodologies will be followed?
  • What hardware and software will be used and how will it be used?
  • What test documentation will be required- See IEEE 829 for more detail on this?

Test Case Development

There are two main components of Test Case Development:

Test Analysis
After test planning is over test analysis phase starts, in this phase we need to dig deeper into project and figure out what testing needs to be carried out in each SDLC phase. Automation activities are also decided in this phase, if automation needs to be done, how much time will it take to automate and which features need to be automated.
Test Design
In this phase various black box and white box testing designs are used to design the test cases for testing, testers start writing test cases by following those design techniques. If automation testing needs to be done, then automation scripts are written in this phase.

Environment Setup

There are two main components of Environment Setup:

Test Construction and Verification
In this case testers prepare more test cases by keeping in mind the positive and negative scenarios, end user scenarios, etc. All test cases and automation scripts needs to be completed in this phase. A Test document plan should be finalized and must be verified by key stakeholders.
Data Setup
Testers create necessary data to execute test cases, preparing for prerequisites defined in the test planning phase

Test Execution and Bug Reporting

Once unit testing is completed by the developers, the build is delivered to the test team and test cases are executed. Bugs are reported during test execution. As developers fix the bugs found, testers re-test and perform regression testing to ensure that the bugs are resolved and that the fixes did not affect any other areas of the software. Test execution reports are created and circulated to the project stakeholders.

After the tester ensures that no more critical bugs remain in the software, the build is delivered to a production mirror environment for a final round of testing. In this phase, the final testing is done for the software, non-functional testing like stress, load and performance testing are performed in this phase. End users also are given a chance to create a test plan to execute cases through their familiar business scenarios to achieve a wider breadth of testing and feedback.

Test Cycle Closure

In this phase, the test and production mirror environments are cleaned and restored to default state, a process review meeting is held and lessons learned are documented for the purpose of continuous improvement.
